# OptimalFit — Native wrapper (Capacitor)

This folder wraps the zero-dependency web app in `..\app\` as native Android and
iOS apps using [Capacitor](https://capacitorjs.com) 8. The web app itself stays
untouched — it is copied here, never edited here.

- **appId:** `com.optimalfit.app` · **appName:** OptimalFit · **webDir:** `www\`
- `server.androidScheme: "https"` (localStorage/data survives, secure context)
- Requires Node.js LTS (installed system-wide: v24.18.0, npm 11.16.0)

## Layout

| Path | What it is |
|------|------------|
| `capacitor.config.json` | Capacitor config (appId, appName, webDir, androidScheme) |
| `scripts\copy-web.js` | Mirrors `..\app` → `www\` and neutralizes `sw.js` (see below) |
| `www\` | GENERATED copy of the web app — never edit, always re-generated by copy-web |
| `android\` | Native Android Gradle project (`npx cap add android` output + our version/label tweaks) |
| `node_modules\` | @capacitor/core, @capacitor/cli, @capacitor/android (8.4.1) |

## Re-sync after any change to the web app

```
cd C:\Users\kjets\optimal-fit\native
npm run sync
```

`sync` = `copy-web` (mirror app → www, wipe-and-copy so deletions propagate)
followed by `npx cap sync` (copies www into
`android\app\src\main\assets\public` and updates native plugins). Run it every
time anything in `..\app\` changes, before building the Android app.

`npm run copy-web` alone just refreshes `www\` without touching the native side.

## The sw.js neutralization (important, do not "fix" it)

The web app registers a service worker (`app\sw.js`, cache-first PWA shell).
Inside the native WebView a service worker is pointless and causes stale-asset
bugs after every sync. We must not edit `app\` (browser/PWA users need the real
worker), so `copy-web` REPLACES `www\sw.js` with a tiny self-unregistering
no-op:

```js
self.addEventListener('install',()=>self.skipWaiting());
self.registration.unregister();
```

`index.html` still registers `sw.js`, but the worker immediately unregisters
itself, so native builds always load fresh assets from the app bundle. Every
copy-web run re-applies this automatically.
